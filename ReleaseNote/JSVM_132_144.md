# JSVM 132 内核升级 144 内核版本差异总结

**注** : 此文档为已识别出的大颗粒非兼容性变更，将持续更新。

## 升级概述

JSVM 内核版本默认从 132 升级到了 144 版本。Chromium 上游社区从 133 到 144 版本共发布 323 项特性，带来了更多 W3C 新特性，以及安全，性能，稳定性的提升。内核升级也会引入不兼容性变更的潜在风险，需要开发者关注并及时适配。

## 132 内核 -> 144 内核带来的收益总结

升级到 144 版本后，开发者可以享受到更现代的语法支持和更高效的执行路径。包括新内核对 V8 沙箱持续加固，修复绕过漏洞，漏洞利用难度显著增加。正则表达式增强，引入了更多的正则特性，提高了复杂文本处理的性能。v8 指针压缩优化，V8 进一步改进了 64 位系统上的指针压缩技术，显著减少了 JavaScript 堆内存的占用。垃圾回收 (GC) 延迟降低，通过改进并发标记 (Concurrent Marking) 和增量回收算法，144 版本的长暂停（Jank）频率更低。
